# Vietnam Reproducibility Account

The replication path has been corrected to use fresh data only.

## What Counts As Replication Data

Replication data means outputs generated by the current PISA pipeline:

- population parquet;
- sources parquet;
- split sparse distance matrix parquets;
- run manifest;
- context map and logs where available.

Fleur's historical notebooks and thesis are used only to understand the algorithmic intent: greedy construction, local search, GRASP, elite pools, and path relinking. The old `.npy` and pickle files are not inputs to the replication.

## Current Modeling Choice

The current 130 stroke facilities are handled as existing coverage. A PISA candidate-grid source layer provides possible new locations. For a given threshold, demand already covered by the existing table sources gets zero incremental weight; candidate selection then maximizes additional covered population.

The reported objective is:

```text
existing covered population + incremental population covered by selected candidates
```

This keeps the optimization instance compatible with the scalable `approximated_tradeoff` `MaxCoverInstance` class while preserving the policy interpretation.
